# DmkWords V3.5 全流程核心业务状态流转图与异常状态处理

> 版本：V3.11（2026-07-20，10大业务域+13异常场景全量对齐）

---

## 一、会员状态流转

### 1.1 用户身份状态
```
未付费用户 → 支付500元 → 观察期会员（30天）
观察期会员 → 30天到期 → 已过期（自动转EXPIRED，生成评估报告）
已过期 → 支付会员费 → 正式会员（年费5400/季度1350/半年2700）
正式会员 → 到期未续费 → 缓冲期（15天，续费9折）
缓冲期 → 续费 → 正式会员
缓冲期 → 超15天 → EXPIRED（可还书/不可新借/音频到期锁/续费解锁不重算借期）
```

注：
- 观察期到期自动转 EXPIRED（定时任务 check_observation_expiry 每日 9:30）
- 季度/半年会员复用 OFFICIAL(2) 状态，通过 Order.type 区分
- 退款前必须无活跃借阅记录（BORROWING/OVERDUE）

### 1.2 异常处理
- 支付成功但状态未更新 → 检查支付回调，重试
- 续费后会员状态未更新 → 定时任务修复

## 二、订单状态流转

### 2.1 订单生命周期
```
创建 → 待支付 → 已支付（30分钟内）
待支付 → 已关闭（30分钟超时，每分钟检查）
已支付 → 退款中（用户申请）
退款中 → 退款成功 / 退款失败
退款失败 → 退款中（重试）
```

### 2.2 异常处理
- 重复支付 → 订单已支付直接返回
- 退款7天未到账 → 定时任务告警

---

## 三、晋级状态流转

### 3.1 晋级生命周期
```
书架借书 → 阅读 → 读完最后一页 → 弹出5道题
答对4/5以上 → 测评通过 → 积分结算 → 实体书待线下扫码归还
答对不到4题 → 未通过 → 20分钟后可重考
测评通过 → 本级累计40本书 → 自动晋级 → 证书
```

### 3.2 异常处理
- 题库为空 → 提示"该书暂无测评题目，完成后请联系管理员"
- 同一本书重复测评 → 不重复计分，但允许再做一次

## 四、想读清单（书架）状态流转

> V3.5 关键变更：从"借阅书架"改为"想读清单/收藏夹"。书架与借阅完全分离，纯粹是"我想读这本书"的收藏行为。不需要押金，没有数量限制。

### 4.1 想读清单流程
```
图书详情页 → 加入想读清单（不限数量）
想读清单中 → 从清单预约借书 → 线下到店取书
想读清单中 → 手动移除 → 清单数量 -1
```

### 4.2 异常处理
- 重复加入同一本书 → 提示"该书已在清单中"
- 从想读清单预约借书时库存不足 → 提示"暂无库存，可到店咨询"
- 积分只计实际读完，与清单操作无关

## 五、观察期状态流转

### 5.1 观察期流程
```
支付500元 → 进入30天观察期 → 正常使用
结束前7/5/3/2/1/0天 → 系统提醒续费
30天到期 → 自动转为EXPIRED，生成评估报告 → 用户支付后转为正式会员
用户中途退出 → 按天数扣费退款
```

### 5.2 观察期报告
- 30天结束时生成PDF报告
- 含：读完书数、单词数、打卡天数、阅读数据
- 带小程序码，可分享

## 六、活动状态流转

### 6.1 活动生命周期
```
管理员创建 → 待发布
发布 → 报名中
到达截止时间 → 报名截止（每5分钟自动迁移）
到达开始时间 → 进行中
到达结束时间 → 已结束
任何阶段 → 已取消（管理员操作）
```

### 6.2 异常处理
- 活动取消 → 自动全额退款，无需用户申请

## 七、图书副本状态流转

### 7.1 BookCopy 生命周期
```
在库 → 线上锁定(预约) → 借出(扫码确认)
借出 → 在库(归还)
借出 → 逾期 → 在库(归还+缴纳罚款)
借出 → 丢失 → 已扣除(扣押金)
在库 → 维修 → 在库(修复完成)
在库 → 报废(不可逆)
```

### 7.2 状态说明

| 状态 | 触发条件 | 说明 |
|------|---------|------|
| 在库 | 新增入库 / 归还 / 维修完成 | 可被预约或扫码借出 |
| 线上锁定 | 用户线上预约成功 | 锁定 offline_available，72 小时后自动释放 |
| 借出 | 到馆扫码确认预约 / 直接线下扫码借书 | 21 天借期开始计时 |
| 逾期 | 超过 21 天未归还 | 音频伴读锁死，需归还并缴纳罚款 |
| 丢失 | 确认丢失 | 扣押金（定价 × 1.5），标记不可再借 |
| 维修 | 损坏送修 | 临时不可借，修好后回到"在库" |
| 报废 | 管理员标记 | 不可逆，总库存减少 |

### 7.3 异常处理
- 扫码时该副本不存在 → 提示"该书未入库，请先扫描入库"
- 扫码时副本已借出 → 提示"该书已被借出，预计归还日期：{date}"
- 扫码时副本在维修 → 提示"该书正在维修中，暂时无法借阅"
- 扫码时副本已报废 → 提示"该书已报废，无法借阅"

## 八、押金状态流转

### 8.1 押金生命周期
```
未付费 → 支付中(用户调起支付)
支付中 → 已付费(支付成功回调确认)
支付中 → 未付费(支付失败/超时)
已付费 → 待审核(用户申请退款，校验通过，需管理员审批)    ← V3.8 新增
待审核 → 退款中(管理员审核通过，发起真实退款)
退款中 → 已退还(退款到账确认)
已付费 → 已扣除(丢书/损坏，扣除押金)
已扣除 → 已付费(补缴押金)
已退还 → 已付费(重新缴纳押金)
```

### 8.2 状态说明

| 状态 | code | 触发条件 | 说明 |
|------|------|---------|------|
| 未付费 | 0 | 新用户 / 押金已退还后 | 不可借书 |
| 支付中 | 5 | 用户调起支付，等待回调确认 | 短暂过渡状态 |
| 已付费 | 1 | 支付 1200 元押金 | 可正常借书 |
| 待审核 | 6 | 用户申请退款，等待管理员审批 | V3.8 新增，需管理员 approve |
| 退款中 | 4 | 管理员审核通过，发起真实退款 | 等待退款到账 |
| 已退还 | 2 | 退款到账确认 | 押金已退回 |
| 已扣除 | 3 | 丢书或损坏 | 扣除图书定价 × 1.5 倍 |

### 8.3 退款校验规则
1. 检查该 child 是否有未归还的 borrow_record → 有则拦截
2. 检查该 child 是否有未缴罚款 → 有则拦截
3. 两项均通过 → 允许退款

### 8.4 异常处理
- 退押金时有未还书 → 拦截，提示"请先归还所有图书"
- 退押金时有未缴罚款 → 拦截，提示"请先缴清罚款"
- 重复支付押金 → 订单已支付直接返回，不重复扣款

## 九、预约状态流转

### 9.1 预约生命周期
```
待取(预约成功，锁定库存) → 已取(到馆扫码确认，转为借阅记录)
待取 → 取消(超时 72 小时自动取消，释放库存)
待取 → 取消(用户手动取消，释放库存)
```

### 9.2 状态说明

| 状态 | 触发条件 | 说明 |
|------|---------|------|
| 待取 | 用户线上预约成功 | 锁定 offline_available 库存，72 小时倒计时 |
| 已取 | 到馆扫码确认 | 释放预约锁，创建 borrow_record，21 天借期开始 |
| 取消 | 超时 72 小时 / 用户手动取消 | 释放锁定库存 |

### 9.3 异常处理
- 预约时库存不足 → 提示"暂无库存，可预约"
- 预约时已有相同书未归还 → 提示"您已借阅此书，请先归还"
- 72 小时超时 → 定时任务自动取消预约，释放库存
- 已取时副本状态异常（维修/报废）→ 拦截，提示"该副本状态异常，请联系工作人员"

## 十、权益转让（Benefit Transfer）

### 10.1 权益转让生命周期

```
家长提交申请 → 待审核 → 运营审核通过/驳回
- 审核通过 → 系统自动执行权益转移（源孩子→目标孩子）
- 审核驳回 → 通知家长并附拒绝理由
```

- 源孩子：观察期或正式会员，无活跃借阅记录
- 目标孩子：不能已是正式会员
- 仅限同一用户下的孩子间转让

### 10.2 异常处理

| 场景 | 处理方式 |
|------|---------|
| 审核中不能重复提交 | 提示"已有待审核的转让申请，请耐心等待" |
| 驳回后可重新提交 | 允许修改后重新申请 |
| 只能转让会员权益（押金不能转让）| 押金独立管理，不随会员转移 |
| 源孩子有未归还图书 | 提示"有未归还图书，请先归还后再转让" |
| 目标孩子已是正式会员 | 提示"目标孩子已是正式会员，无需转让" |
| 源孩子状态不允许转让 | 拒绝操作 |

## 十一、V3.5 OMO 核心异常处理汇总

### 10.1 借阅异常

| 场景 | 处理方式 |
|------|---------|
| 借阅时库存不足 | 提示"暂无库存，可预约" |
| 借阅时押金未付 | 提示"请先缴纳押金 ¥1200" |
| 借阅时已达上限（由 borrow_limit 配置决定，默认20本）| 提示"已达最大借阅数，请先归还部分图书" |
| 扫码时副本不存在 | 提示"该书未入库，请先扫描入库" |

### 10.2 归还异常

| 场景 | 处理方式 |
|------|---------|
| 归还时逾期 | 显示逾期天数 + 罚款金额 |
| 归还时图书损坏 | 提示"图书损坏，需缴纳维修费" |
| 归还时图书丢失 | 扣除押金（定价 × 1.5 倍），标记副本为"丢失" |

### 10.3 押金异常

| 场景 | 处理方式 |
|------|---------|
| 退押金时有未还书 | 拦截，提示"请先归还所有图书" |
| 退押金时有未缴罚款 | 拦截，提示"请先缴清罚款" |
| 重复支付押金 | 订单已支付直接返回，不重复扣款 |

### 10.4 预约异常

| 场景 | 处理方式 |
|------|---------|
| 预约时库存不足 | 提示"暂无库存，可预约" |
| 预约时已有相同书未归还 | 提示"您已借阅此书，请先归还" |
| 72 小时超时未取 | 定时任务自动取消，释放库存 |
| 已取时副本状态异常 | 拦截，提示"该副本状态异常，请联系工作人员" |
