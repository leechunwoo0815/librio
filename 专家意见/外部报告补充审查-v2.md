### 🔴 遗漏 1: `seed_default_configs()` 键名与代码不匹配（报告未发现）

**严重程度: P1** — 管理端修改价格不生效（该 bug 在报告 §"配置种子缺失" 中被遗漏）。

#### 证据

**Seeder** (`backend/seeds/seed_rbac.py:300-324`):
```python
defaults = {
    "observation_price": ("500.00", "decimal", "观察期价格"),      # ← 注意键名
    "official_member_price": ("5400.00", "decimal", "正式会员价格"), # ← 注意键名
    ...
}
```

**代码读取** (`backend/domain/order/service.py:82-83`):
```python
OrderType.OBSERVATION: "price_observation",           # ← 不同的键名！
OrderType.OFFICIAL_MEMBER: "price_official_member",   # ← 不同的键名！
```

**影响链路**:
1. Seeder 写入 DB: `config_key="observation_price"`, `config_value="500.00"`
2. 代码查询 DB: `ConfigService.get_decimal(db, "price_observation", Decimal("500"))`
3. DB 中查不到 `"price_observation"` → 走 fallback `Decimal("500")` → **功能正常但永远用不到 DB 值**
4. 管理员在后台修改价格 → 写入 `"price_observation"` → 这条写入才有效
5. Seeder 写入的 `"observation_price"` 是**死数据**

**修复方案**:
```python
# backend/seeds/seed_rbac.py:302-311
defaults = {
    "price_observation": ("500.00", "decimal", "观察期价格"),         # 修正键名
    "price_official_member": ("5400.00", "decimal", "正式会员年费"),  # 修正键名
    # ... 其余不变
}
```

## 四、报告完全未覆盖的发现

### 新增 P1: 报告自身的 `seed_default_configs` 键名 Bug

已在上方 §遗漏 1 详述。这里补充影响分析：

**触发条件**: 管理员在后台"系统配置"页面修改"观察期价格"或"正式会员年费"。

**实际行为**: 
- 如果 seeder 先运行（写入 `observation_price`），管理员后通过 UI 修改 → 写入了正确的 `price_observation` → **功能正常**（因为 `ConfigService` 查 `price_observation` 能找到 DB 值）
- 但如果一个全新的部署，seeder 写了 `observation_price`，管理员还没改过 → `ConfigService` 查 `price_observation` 找不到 → fallback 到 DEFAULTS → **功能正常但用的是代码硬编码值**

**根因**: seeder 键名与代码读取键名不匹配。不是致命 bug（有 fallback），但应修复以避免管理端价格修改的行为困惑。
