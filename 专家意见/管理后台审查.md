## 二、核心发现

### ⚠️ P1：levels.html 表单缺 loading 防重复提交

**文件**: `backend/templates/admin/levels.html`

单表单 `onsubmit="window.levelsPage.submitLevel(event)"`，但在模板中未发现 `setAttribute('disabled')` 或 loading 状态切换。用户双击可触发两次 POST。

**修复**: 在 `submitLevel` 函数开头加：
```javascript
const btn = document.querySelector('#levelForm button[type="submit"]');
btn.disabled = true;
btn.textContent = '提交中...';
// finally 中恢复 btn.disabled = false;
```

---

### ⚠️ P1：users.html 24 个全局函数未外迁

**文件**: `backend/templates/admin/users.html`

这是管理后台**最复杂的单页模板**——24 个全局函数混在 `<script>` 标签中，包括用户 CRUD、孩子 CRUD、弹窗管理、表单提交、CSV 导出。是上一个重构轮次遗留的最大未外迁模板。

**影响**: 全局命名空间污染、难以测试、难以复用。

**建议**: 参考已完成的 `books.html→pages/books.js` 模式，将 users.html 的 JS 外迁到 `pages/users.js`。工作量约 2 小时。

---

### ⚠️ P2：3 个模板的 form onsubmit 使用 `return` 而非 `e.preventDefault()`

| 文件 | 模式 | 风险 |
|------|------|------|
| `bookcopy.html` | `onsubmit="return addItem(event)"` | 与主流 `e.preventDefault()` 不一致 |
| `message_manage.html` | `onsubmit="return sendMessage(event)"` | 同上 |
| `achievements.html` | `onsubmit="submitBadge(event)"` | 虽传了 event 但内部未调 preventDefault |

**实际上不会有功能 bug**——因为这些函数都通过 `try/catch` 处理了异常且没有页面跳转行为。但这属于代码风格不统一。

---

### ✅ 全局 fetch 无滥用

仅 2 处非共享 API 的 fetch：

**其余所有 API 调用均通过 `App.api.get/post/put/del` 统一层。** 此前担心的裸 fetch 问题不存在。
