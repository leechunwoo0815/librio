## 第一部分：并发锁批量修复（P1-1 ~ P1-15）

### 修复模板

所有状态变更操作的对象查询必须加 `.with_for_update()`，模板如下：

```python
# 修复前（反例）
obj = self.some_repo.get_by_id_or_raise(obj_id)

# 修复后
obj = (
    self.db.query(SomeModel)
    .filter(SomeModel.id == obj_id, SomeModel.is_deleted == 0)
    .with_for_update()
    .first()
)
if not obj:
    raise NotFoundError("对象不存在")
```

#### P1-15. book/reading/child 等 19 领域 — 系统性加锁

这 19 个领域 service 目前零 `with_for_update()`。按优先级分层修复：

**第一层（涉及状态变更，必须修）**:
- `book/service.py` — `create_book`/`update_stock` 等库存操作
- `reading/service.py` — `save_progress`/`end_session` 进度写入
- `child/service.py` — `update_status`/`update_deposit` 状态变更
- `activity/service.py` — 活动报名状态变更

**第二层（纯查询/创建，可选）**:
- 其余 15 个领域，如果没有并发写入场景，可以不加

**修复原则**：任何"先读后写"的模式都需要 `with_for_update()`。纯 `INSERT` 不需要。
