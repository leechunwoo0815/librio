## 二、核心发现

### ✅ 已确认优秀项：共享 network 层

`utils/request.js`（110 行）是正面典范：

- **网络检测前置**（L23-38）: `wx.getNetworkType` → 无网络时直接 reject + toast
- **401 自动重登**（L59-73）: 清 token + reLaunch 到 login，且排除了 login 页面自身的 401
- **403 权限提示**（L76-79）: toast "无权限"
- **统一 fail 回调**（L93-100）: 网络异常 toast
- **超时设置**（L55）: POST/PUT 15s, GET 10s
- **showError 控制**（L87-89）: 支持静默模式

**所有页面通过 `utils/api.js` 调用 `request.js`，集中式错误处理无需各页面重复。此前扫描发现的 10 个页面"缺 fail/catch"实为假阳性——均已被共享层覆盖。**

### ⚠️ P1：observation-report.wxml 裸属性访问

**文件**: `frontend/pages/member-pkg/observation-report/observation-report.wxml`

```xml
{{report.observation_start}}  <!-- ❌ report 为 null 时崩溃 -->
```

**对比其他页面的正确做法**:
```xml
{{child.name || '学员'}}      <!-- ✅ 有 fallback -->
{{summary.total_minutes || 0}} <!-- ✅ 有 fallback -->
```

**风险**: 报告接口返回 null 或网络异常时，页面白屏崩溃。

**修复**: 改为 `{{report.observation_start || '加载中...'}}` 或外层加 `wx:if="{{report}}"`。

### ⚠️ P2：5 页定时器未在 onUnload 清除

| 页面 | 定时器 | 影响 |
|------|--------|------|
| `benefit-transfer.js:141` | `setTimeout(() => wx.navigateBack(), 1500)` | 页面已离开仍触发导航 |
| `certificate.js:311` | `setTimeout(download, 1000)` | 下载回调在已销毁页面触发 |
| `observation.js:98` | `setTimeout(() => {...})` | 同上 |
| `official.js:134` | `setTimeout(() => {...})` | 同上 |
| **`refund-apply.js:112`** | `this._lastCalcTimer = setTimeout(...)` | **已存引用但从未清除** |

`refund-apply` 是最严重的——它存储了 timer 引用但 `onUnload` 中完全没有 `clearTimeout(this._lastCalcTimer)`。其他 4 页虽不存引用，但至少应加 `onUnload` 清理。

**修复模板**:
```javascript
onUnload() {
  if (this._lastCalcTimer) { clearTimeout(this._lastCalcTimer); this._lastCalcTimer = null; }
  if (this._navTimer) { clearTimeout(this._navTimer); this._navTimer = null; }
}
```
