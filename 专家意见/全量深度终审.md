## 六、事件处理器异常模式全量审计

逐 handler 检查异常处理：

| handler | 异常处理 |
|---------|---------|
| `handle_book_borrowed_for_copy_status` | ⚠️ 无 try/except，异常传播到主事务回滚 |
| `handle_book_returned_for_copy_status` | ✅ try/except + logger.error + raise |
| `handle_reservation_*` | ⚠️ 无 try/except，但操作简单（库存增减） |
| `handle_order_paid` | ✅ 事务内，子状态校验完整 |
| `handle_deposit_paid_for_child` | ⚠️ 只更新 child.deposit_status，无 try/except |

### ⚠️ 共享事务模式下 handler 异常的行为

当 `publish(db=session)` 时，handler 在同一个事务中执行。如果 handler 抛出异常，**整个事务回滚**，包括 publisher 的所有操作。

**这意味着**:
- `borrow_book()` 中，如果 `handle_book_borrowed_for_copy_status` 失败，借阅记录也被回滚——用户借书操作整个失败
- 这是 DDD 的预期行为，但需要确保 handler 不会因为非关键操作失败而影响核心流程

## 修复提示

7 个 WXML 页面的修复是**完全相同的模板**——在数据容器外层加一个 `wx:if`：

```xml
<!-- 修复前 -->
<view class="content">{{object.field}}</view>

<!-- 修复后 -->
<view class="content" wx:if="{{object}}">{{object.field}}</view>
<view class="empty-state" wx:else>加载中...</view>
```

所有 7 页可以**批量修改**，不需要逐个写 diff。
