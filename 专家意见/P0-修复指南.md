## P0-1. audit_refund 支付网关失败被吞 → 假退款

**文件**: `backend/domain/deposit/service.py`

**问题代码** (行 379-387):
```python
if payment_gateway:
    try:
        payment_gateway.refund(                                    # 如果这里抛异常
            order_no=str(record.pay_order_id) if record.pay_order_id else "",
            amount=record.amount,
            reason="押金退款（审核通过）",
        )
    except Exception as e:                                          # 被这里捕获
        logger.error(f"Refund API call failed (non-blocking): ...") # 仅记日志
# ↓ 继续执行，commit 了 REFUNDING 状态，但钱没退！
```

**修复为**:
```python
if payment_gateway:
    try:
        result = payment_gateway.refund(
            order_no=str(record.pay_order_id) if record.pay_order_id else "",
            amount=record.amount,
            reason="押金退款（审核通过）",
        )
        # 检查退款结果（如果 gateway 返回结果对象）
        if hasattr(result, 'success') and not result.success:
            raise PaymentError(getattr(result, 'error_message', '退款接口返回失败'))
    except Exception as e:
        self.db.rollback()                                         # ← 回滚数据库
        logger.error(f"Refund failed, transaction rolled back: child={child_id}, error={e}")
        raise PaymentError(f"押金退款调用失败: {e}")              # ← 重新抛出，让调用方感知
```

**变更要点**:
1. 异常捕获后先 `self.db.rollback()`，确保 REFUNDING 状态不持久化
2. 用 `raise` 重新抛出异常，让 router 层返回 500/错误响应给管理员
3. 检查 `result.success`（如果 gateway 实现了返回值检查）

## P0-2. refund audit_refund 缺并发锁 → 可双重审核

**文件**: `backend/domain/refund/service.py`

**问题代码** (行 116-120):
```python
def audit_refund(self, refund_id: int, audit: RefundAudit) -> RefundResponse:
    """审核退款"""
    refund = self.refund_repo.get_by_id_or_raise(refund_id)  # ← 无 with_for_update()
    if refund.status != RefundApplication.STATUS_PENDING:
        raise ConflictError("申请已处理")
```

**修复为**:
```python
def audit_refund(self, refund_id: int, audit: RefundAudit) -> RefundResponse:
    """审核退款 — 带行锁防止双重审批"""
    refund = (
        self.db.query(RefundApplication)
        .filter(RefundApplication.id == refund_id, RefundApplication.is_deleted == 0)
        .with_for_update()                                       # ← 加锁
        .first()
    )
    if not refund:
        raise NotFoundError("退款申请不存在")
    if refund.status != RefundApplication.STATUS_PENDING:
        raise ConflictError("申请已处理")
    ...
    # Order 查询同样需要加锁（行 130）
    order = (
        self.db.query(Order)
        .filter(Order.id == refund.order_id, Order.is_deleted == 0)
        .with_for_update()                                       # ← 加锁
        .first()
    )
```

**变更要点**:
1. RefundApplication 查询改用 `self.db.query(...).with_for_update().first()`
2. Order 查询同样加 `with_for_update()`
3. 不能用 `get_by_id_or_raise` — 它不提供行锁，改用显式 query
