## P2-1. scan_and_borrow 库存更新风格不一致

**文件**: `backend/domain/borrow/service.py:253-254`

```python
# 当前（Python 算术）
book.total_stock = (book.total_stock or 0) + 1
book.available_stock = (book.available_stock or 0) + 1

# 建议改为（SQL 原子更新，与 borrow_book 风格一致）
self.db.query(Book).filter(Book.id == book.id).update(
    {Book.total_stock: Book.total_stock + 1, Book.available_stock: Book.available_stock + 1},
    synchronize_session='fetch'

```

**优先级**: 低。当前场景是新建书（无并发写入），但风格统一有利于维护。

## P2-4. 事件总线独立 session 模式下重试 session 创建异常

**文件**: `backend/common/events.py:398-403`

```python
# 当前
retry_session = get_session()()
handler(event, retry_session)  # 如果 get_session()() 失败，retry_session 未定义

# 建议
try:
    retry_session = get_session()()
except Exception as create_err:
    logger.error(f"Failed to create retry session: {create_err}")
    self._record_dead_letter(event, handler.__name__, str(create_err))
    continue
handler(event, retry_session)
```

**优先级**: 极低。触发条件概率极低（连接池耗尽 + 首次失败 + 重试时仍然耗尽）。

## 系统级改进建议（非具体 Bug）

### 1. 引入 `with_for_update` lint 规则

在 CI 中增加检查：所有 `def.*update\|def.*delete\|def.*cancel\|def.*return\|def.*deduct\|def.*refund` 方法中必须有 `with_for_update()` 调用。

### 2. 统一 Repository 模式

建议 BaseRepository 增加 `get_by_id_for_update()` 方法，统一加锁逻辑：

```python
class BaseRepository:
    def get_by_id_for_update(self, id: int):
        return self.db.query(self.model).filter(
            self.model.id == id, self.model.is_deleted == 0
        ).with_for_update().first()
```

### 3. Prometheus 监控端点

当运维体系就绪后，安装 `prometheus-fastapi-instrumentator` 暴露 `/metrics`。
