### F1. 年卡页面显示 ¥NaN

**复现路径**: 家长打开「开通年卡」页面 → 看到 `¥NaN` 价格

**用户视角**: 价格显示 `¥NaN`——家长以为遇到骗子，直接退出小程序。品牌信任归零。

**Root Cause**: `official/official.js:75-78` — `loadChild()` 中从 `priceMap[periodType]` 取价格，当 API 返回的 tier 数据缺少对应 `periodType` 键时，`priceMap[periodType]` 为 `undefined`，后续 `Math.round(undefined / 0.9)` 产生 `NaN`，直接渲染到 WXML。

```javascript
// official.js:75-78 — 脆弱代码
const actualPrice = priceMap[periodType];          // undefined if key missing
const origPrice = Math.round(actualPrice / 0.9);   // NaN
const savings = origPrice - actualPrice;            // NaN
const secondChildPrice = Math.round(actualPrice * 0.9); // NaN
```

**Fix** — `official.js:74-80`:
```javascript
const rawPrice = priceMap[periodType];
const actualPrice = (rawPrice != null && !isNaN(rawPrice)) ? Number(rawPrice) : 0;
const origPrice = actualPrice > 0 ? Math.round(actualPrice / 0.9) : 0;
const savings = origPrice - actualPrice;
const secondChildPrice = actualPrice > 0 ? Math.round(actualPrice * 0.9) : 0;
```

**同模式修复**: `switchPeriod()` 函数第 98-102 行完全相同的算式，同样的 fix。

**连坐**: `order-history/order-history.js:70` — `Number(order.amount).toFixed(2)` 当 amount 为 `undefined` 时产生 `"NaN"`：
```javascript
// 改前
amountText: Number(order.amount).toFixed(2),
// 改后
const amtNum = Number(order.amount);
amountText: (order.amount != null && !isNaN(amtNum)) ? amtNum.toFixed(2) : '0.00',
```

### F2. 做完 5 道题提交 → 跳白屏

**复现路径**: 孩子完成 5 道测验题 → 点击提交 → 跳转到白屏页面

**用户视角**: 孩子做完题，期待看到分数和鼓励，结果白屏——孩子以为做错了，哭着找家长。

**Root Cause 1（直接原因）**: `quiz-result.wxml:6` — 主内容区靠 `total > 0` 守卫，但当 `options.total` 缺失时（`parseInt(undefined)` → `NaN` → `0`），条件失败，且 `loadError=false` + `loading=false` 导致三种状态全部隐藏，页面空白。

```html
<!-- quiz-result.wxml:6 — 脆弱代码 -->
<block wx:if="{{!loadError && !loading && total > 0}}">
  <!-- 无 total ≤ 0 时的 fallback！ -->
</block>
```

**Root Cause 2（深层原因）**: Quiz 提交 API 的 request/response schema 全部不匹配，实际 HTTP 400/500，导致前端拿不到数据：

- **Request**: 前端发 `{"answers": [{"question_id": 1, "answer": "A"}]}`，后端 schema 要求 `quiz_id` + `selected_answer` → HTTP 422
- **Response**: service 返回 `{"correct", "total", "score"}`，router 声明 `response_model=QuizResultResponse` 但 schema 只定义了 `quiz` 字段 → HTTP 500

**Fix 1** — `quiz-result.wxml:6` 加 fallback:
```html
<block wx:if="{{!loadError && !loading && total > 0}}">
  <!-- existing main content -->
</block>
<view wx:elif="{{!loadError && !loading && total === 0}}" class="empty-state">
  <text class="empty-emoji">📝</text>
  <text>暂无测评数据</text>
  <button bindtap="goBackToShelf" class="btn-primary">返回书架</button>
</view>
```

**Fix 2** — 对齐 Quiz submit API schema（选 Option A: 改 schema 匹配 service）:
```python
# schemas.py:64-67 — QuizResultResponse
class QuizResultResponse(BaseSchema):
    correct: int
    total: int
    score: float
    passed: bool
    word_count: int = 0
```

**Fix 3** — 前端 quiz.js 修正请求体字段名:
```javascript
// quiz.js:220-223 — 改前
const answerArray = questions.map(q => ({
    question_id: q.id,
    answer: answers[q.id] || ''
}))
// 改后
const answerArray = questions.map(q => ({
    quiz_id: quizId,
    question_id: q.id,
    selected_answer: answers[q.id] || ''

```

### F3. 打开一本书只有英文文本，没音频，没提示

**复现路径**: 孩子打开一本书 → 看到英文文本 → 没有声音播放，没有错误提示

**用户视角**: 孩子不认识单词，等声音帮助，但什么都没发生——不知道怎么读，不知道是坏了还是就是没有。

```javascript
}
// 如果 has_audio=true 但 audio_url=null: initAudio() 被跳过，UI 仍显示
```

**同时**: `reader.js:203-217` — `initAudio()` 设置了 `bgAudioManager.src` 但从未调用 `bgAudioManager.play()`，即使有音频也不会自动播放。

**Root Cause — 后端**: `backend/domain/book/service.py:107-115` — `get_book_detail()` 查询了 `AudioFile` 但只取 `reader` 和 `duration_seconds`，从未将 `audio.file_url` 赋值给 `resp.audio_url`，导致前端永远收不到音频 URL。

```python
    resp.audio_duration = audio.duration_seconds
```

**Fix 1** — 前端: 分离播放器渲染条件和音频缺失提示:

```javascript

// 改后
if (book && book.has_audio) {
```

### F4. 进入孩子管理页 → 白屏崩溃 `null.find()`


**Fix 1** — JS 侧加 `|| []`:

### S3. 支付回调丢失 → 付了钱状态不变

**复现路径**: 家长微信支付成功 → 回到小程序 → 订单仍显示「未支付」→ 重复支付 / 投诉

**用户视角**: 钱扣了但状态没变——家长恼火投诉是诈骗，管理员需要手动改数据库。

**Root Cause 1** — `order/router.py:164-168`: `OrderPayCallback` 构造时**缺少 `amount`**（必填字段，`Field(...)`），Pydantic 校验失败返回 HTTP 422：

```python
# router.py:164-168 — 脆弱代码
callback = OrderPayCallback(
    order_no=decrypted.out_trade_no,
    trade_no=decrypted.transaction_id,
    pay_type=1,
    # ❌ amount 缺失 → Pydantic 422 回调永远失败
)
```

**Root Cause 2** — `pay_v3.py:241`: 微信 V3 回调金额以**分**为单位，系统**从未转换为元**：

```python
# pay_v3.py:241 — 脆弱代码
amount=data.get("amount", {}).get("total"),
# total=5000（表示 ¥50.00），但系统当作 ¥5000 处理
```

即使 root cause 1 修复，`service.py:228` 的金额比较也会失败：
```python
if callback.amount != order.amount:  # 5000 != 50.00 → PaymentError
```

**Fix 1** — 回调构造传入 amount 并转换单位:
```python
# router.py:164-168 — 改前
callback = OrderPayCallback(
    order_no=decrypted.out_trade_no,
    trade_no=decrypted.transaction_id,
    pay_type=1,
)

# 改后
from decimal import Decimal
callback = OrderPayCallback(
    order_no=decrypted.out_trade_no,
    trade_no=decrypted.transaction_id,
    pay_type=1,
    amount=Decimal(str(decrypted.amount)) / Decimal("100")
    if decrypted.amount is not None else Decimal("0"),
)
```

**Fix 2** — `pay_v3.py` decrypt 时转换分到元:
```python
# pay_v3.py:241 — 改前
amount=data.get("amount", {}).get("total"),
# 改后
amount_raw = data.get("amount", {}).get("total")
amount = Decimal(str(amount_raw)) / Decimal("100") if amount_raw is not None else None
```

**同模式修复** — 押金回调也需要分转元:
```python
# deposit/router.py:104 — 改前
callback_amount = Decimal(str(callback_data.amount)) if callback_data.amount is not None else None
# 改后
callback_amount = (Decimal(str(callback_data.amount)) / Decimal("100")) \
    if callback_data.amount is not None else None
```
