**文件**: `backend/templates/admin/levels.html`
**问题**: `<form onsubmit="window.levelsPage.submitLevel(event)">` — 双击可触发两次 POST

**当前代码** (`submitLevel` 函数):
```javascript
submitLevel(event) {
  event.preventDefault();
  // ...表单校验 + App.api.post ...
}
```

**修复**: 函数开头加 loading 锁，finally 恢复：
```javascript
submitLevel(event) {
  event.preventDefault();
  const btn = document.querySelector('#levelForm button[type="submit"]');
  if (btn.disabled) return;           // 防重复
  btn.disabled = true;
  btn.textContent = '提交中...';

  try {
    // ... 现有表单校验 + App.api.post 逻辑 ...
  } finally {
    btn.disabled = false;
    btn.textContent = '保存';
  }
}
```

## 3. 第三优先级 — P2（1 项管理端风格统一）— ✅ 已完成

### P2-6. 3 模板 onsubmit 风格统一

**文件**:
| # | 文件 | 行号 | 当前模式 |
|---|------|------|---------|
| 1 | `bookcopy.html` | ~行1 | `<form onsubmit="return addItem(event)">` |
| 2 | `message_manage.html` | ~行1 | `<form onsubmit="return sendMessage(event)">` |
| 3 | `achievements.html` | ~行1 | `<form onsubmit="submitBadge(event)">` |

**问题**: 管理端主流模式是 `onsubmit="return handler(event)"`，但部分页面用 `onsubmit="handler(event)"` 无 `return`，部分页面的 handler 内部未调 `event.preventDefault()`。

**修复**: 统一为管理端标准模式：
```html
<!-- 改前 -->
<form onsubmit="return addItem(event)">
<!-- 改后 -->
<form onsubmit="addItem(event);return false">
```

同时在对应 JS 函数开头加：
```javascript
function addItem(event) {
  event.preventDefault();
  // ... 现有逻辑 ...
}
```

> 注意：`bookcopy.html` 和 `messages.html` 引用的 JS 在 `static/admin/js/pages/` 下，`achievements.html` 的 JS 在模板内联中。分别修改。
