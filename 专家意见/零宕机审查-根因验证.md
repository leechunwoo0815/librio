## F1 — 年卡页面显示 ¥NaN

**严重度**: 致命 | **文件**: `frontend/pages/order-pkg/official/official.js:75-78,98-102` | **波及**: 年卡/观察期/历史订单 3 页

### 根因

`loadChild()` 中 `priceMap[periodType]` 返回 `undefined`（API 返回的 tier 数据缺少对应周期键），后续 `Math.round(undefined / 0.9)` 产生 `NaN`，直接渲染到 WXML 显示 `¥NaN`。`switchPeriod()` 第 98-102 行完全相同的算式。

### 验证代码

```javascript
// official.js:75-78 — 已验证
const actualPrice = priceMap[periodType];          // undefined if key missing
const origPrice = Math.round(actualPrice / 0.9);   // NaN
const savings = origPrice - actualPrice;            // NaN
const secondChildPrice = Math.round(actualPrice * 0.9); // NaN
```

### 修复

```javascript
// official.js:74-80 — loadChild()
const rawPrice = priceMap[periodType];
const actualPrice = (rawPrice != null && !isNaN(rawPrice)) ? Number(rawPrice) : 0;
const origPrice = actualPrice > 0 ? Math.round(actualPrice / 0.9) : 0;
const savings = origPrice - actualPrice;
const secondChildPrice = actualPrice > 0 ? Math.round(actualPrice * 0.9) : 0;

// official.js:98-102 — switchPeriod() 完全相同的修复
```

### 连坐修复

`frontend/pages/order-pkg/order-history/order-history.js:70` — 同样产生 `"NaN"`：

